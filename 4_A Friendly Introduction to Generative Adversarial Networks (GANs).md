## GAN 的核心思想

GAN 由**一对相互对抗的神经网络**组成：

- **生成器（Generator）**
- **判别器（Discriminator）**

它们的行为很像**造假者**和**警察**：

- 造假者不断尝试制作假画 → 对应**生成器**
- 警察不断尝试抓住造假者 → 对应**判别器**

每当造假者被警察抓住，他就会不断改进自己的画作，直到有一天，他终于画出一幅完美的赝品，彻底骗过了警察。

**用神经网络的语言来说**：我们有一对神经网络——生成器和判别器——并用一组**真实图像**和一组由生成器产生的**假图像**来训练它们。

- **判别器**被训练去识别哪些图像是真实的、哪些来自生成器。
- **生成器**被训练去欺骗判别器，让判别器把它生成的图像判定为真实图像。

## 设定：Slanted Land（倾斜之地）

为了便于讲解，我们会构建一对极其简单的 GAN，简单到可以直接用 Python 编写，**完全不需要任何深度学习框架**。

设定如下：我们生活在一个叫做 **Slanted Land（倾斜之地）** 的世界，这里的所有人看起来都略微被拉长，并且以 45 度角行走。这个世界有技术，但不如我们发达：

- 他们有显示图像的电脑屏幕，但能做到的最高分辨率只有 **2×2**，即黑白的 **2 像素 × 2 像素** 屏幕。
- 他们也发明了神经网络，但只会**单层**的神经网络。

我将向你展示：即便在这个简单的世界里，我们依然能构建一对 GAN，生成 Slanted Land 居民的人脸。

### 人脸长什么样

Slanted Land 的居民都是被拉长的、向左倾斜 45 度的。由于屏幕只有 2×2 像素，他们的人脸在屏幕上看起来就像一条**对角线**（类似反斜杠 `\`）。

而**噪声图像**则不一样——它们不像反斜杠，大多是随机生成的，因此不是人脸。

**网络的目标**就是把这样的人脸图像，与噪声图像（非人脸）区分开来。

### 给像素赋值

我们用一个数值刻度：

- **白色像素 = 0**
- **黑色像素 = 1**

（你可能见过相反的约定，但这里为清晰起见就这么定。）这样我们就能给 2×2 屏幕中的每个像素赋一个数值。

## 构建判别器（Discriminator）

先来手工构建判别器。第一个问题是：**怎么把人脸和非人脸区分开？**

观察发现：在人脸中，**左上角**和**右下角**的像素颜色较深，所以数值较大；而**另外两个角**颜色较浅，数值较小。而在噪声图像里，任何情况都可能发生。

因此区分人脸的方法是：

> 把**左上角 + 右下角**的值相加，再**减去另外两个角**的值。

对人脸来说，这个结果会很大；对噪声图像来说，结果会很小。

**举例**：

- 某张人脸：得分 = **2**
- 某张噪声图：得分 = **−0.5**

我们设一个**阈值（threshold）= 1**：

- 得分 **≥ 1** → 判定为人脸
- 得分 **< 1** → 判定为非人脸（噪声图像）

### 用神经网络的语言描述

- 4 个像素的值分别乘以 **+1 或 −1**（取决于它们在哪条对角线上）。
- 然后减去一个总值 1，即**偏置（bias）**。
- 把这 4 个数加起来得到得分：得分 ≥ 1 判为人脸，< 1 判为非人脸。

### 用 Sigmoid 给出概率

我们还可以用 **sigmoid 函数**把得分转换成"是人脸的概率"。sigmoid 会把大的数映射到接近 1 的值，把小的数映射到接近 0 的值。

**示例 1（人脸）**：

- 得分 = 1
- sigmoid(1) ≈ **0.73**
- 判别器认为这张图有 **73%** 的概率是人脸。由于大于 50%，判定为人脸 → **正确**。

**示例 2（非人脸）**：

- 得分 = −0.5
- sigmoid(−0.5) ≈ **0.37**
- 低于 50%，判定为非人脸 → **正确**。

> **约定**：在神经网络图中，**粗边表示正权重**，**细边表示负权重**。本视频全程使用这一约定。

## 构建生成器（Generator）

生成器是一个会**输出人脸**的神经网络。同样地，我们利用这一规律：人脸中左上、右下两角数值大，另外两角数值小；而噪声图像里则什么都可能。

生成器的工作流程：

1. **选取输入 z**：一个 0 到 1 之间的随机数（示例中取 **0.7**）。一般来说，输入是来自某个固定分布的向量。
2. **构建网络权重**：我们希望输出有的大、有的小。大的用粗边表示，小的用细边表示。
   - 因为想让**左上角和右下角**输出大值 → 对应权重设为 **+1**
   - 想让**右上角和左下角**输出小值 → 对应权重设为 **−1**

**计算得分（以 z = 0.7 为例）**：

| 像素位置 | 权重 | 得分计算 | 得分 |
|---|---|---|---|
| 左上 | +1 | (+1)(0.7) + 1 | **1.7** |
| 右上 | −1 | (−1)(0.7) − 1 | **−1.7** |
| 左下 | −1 | (−1)(0.7) − 1 | **−1.7** |
| 右下 | +1 | (+1)(0.7) + 1 | **1.7** |

**应用 sigmoid 得到像素值**：

- sigmoid(1.7) ≈ **0.85**（左上、右下）
- sigmoid(−1.7) ≈ **0.15**（右上、左下）

这些就是放进各像素的值。结果图像看起来像一条**对角线**，正符合我们对人脸的定义。

> 由于输入 z 始终在 0 到 1 之间，无论输入什么，这个网络**总会生成左上/右下大、右上/左下小**的图像，也就是**总会生成人脸**。因此它是一个好的生成器。

当然，上面这些权重是我们**靠眼力估出来的**，实际中并非如此——一般需要**训练**神经网络来得到最优权重。下面就介绍训练所需的工具。

---

## 误差函数（Error / Cost Function）

误差函数（或称代价函数）是告诉网络"它做得怎么样"的方式，以便改进：

- 误差大 → 网络表现差，需要降低误差来改进。

我们训练 GAN 用的误差函数是 **log loss（对数损失）**，即**自然对数**（以 e 为底）。对数之所以频繁出现在误差函数中，有很多更深层的原因，这里不展开，可以先把它当作一个非常方便的函数。

### 情形一：标签为 1（希望预测接近 1）

- 预测 0.1 → 很糟糕（离 1 很远）→ 误差应大
- 预测 0.9 → 很好（接近 1）→ 误差应小

使用 **−ln(预测值)**：

- −ln(0.1) ≈ **2.3**（大）
- −ln(0.9) ≈ **0.1**（小）

预测越接近 1，−ln 越小。所以当标签为 1 时，**误差 = −ln(预测值)** 是个好的误差函数。

### 情形二：标签为 0（希望预测接近 0）

- 预测 0.1 → 很好（接近 0）→ 误差应小
- 预测 0.9 → 很糟糕 → 误差应大

使用 **−ln(1 − 预测值)**：

- 预测 0.1：−ln(0.9) ≈ **0.1**（小）
- 预测 0.9：−ln(0.1) ≈ **2.3**（大）

正符合我们的期望。

### 小结

| 标签 | 希望预测 | 误差函数 |
|---|---|---|
| 1 | 接近 1 | **−ln(prediction)** |
| 0 | 接近 0 | **−ln(1 − prediction)** |

从 −ln(x) 的图像看：预测接近 0 时误差大、接近 1 时误差小。
从 −ln(1−x) 的图像看：预测接近 0 时误差小、接近 1 时误差大。

这两个误差函数将分别用于训练生成器和判别器，取决于我们希望预测结果是 0 还是 1。

---

## 反向传播（Backpropagation）

有了误差函数，就进入训练的核心——**反向传播**。简要说明训练流程：

1. **前向传播（forward pass）**：取一个数据点，计算出预测值。
2. **计算误差**：基于前面定义的 log loss。
3. **求导**：用**链式法则**，对所有权重求误差的导数。这告诉我们每个权重该如何调整，才能最大程度地降低误差。
4. **梯度下降（gradient descent）**：
   - 计算**梯度**——误差增长最快的方向。
   - 沿梯度的**反方向**走一小步，得到能降低误差的新参数。

---

## 训练 GAN：把误差函数放对位置

现在开始训练生成器和判别器。关键是把**正确的误差函数放在正确的位置**。

流程如下：

1. 初始时网络权重**未定义**，先设为**随机数**。
2. 选取一个 0–1 之间的随机数作为生成器的输入 z。
3. 对生成器做**前向传播**，得到一张图像（由于权重随机，这通常**不是**人脸）——这就是**生成图像**。
4. 把生成图像送入**判别器**，判断真假。判别器输出一个概率，比如 **0.68**。

接下来是关键——**定义正确的误差函数**：

### 判别器想要什么？

由于这张图是生成器造的假图，一个优秀的判别器**应该判定它为假**，即**输出 0**。

当我们希望网络输出 0 时，误差函数是：

> **判别器误差 = −ln(1 − prediction)**

这个误差用来训练**判别器**。

### 生成器想要什么？

生成器的终极梦想是：生成一张足够逼真的图，**骗过判别器**，让判别器把它判为真（**输出 1**）。

当我们希望（整个串联网络）输出 1 时，误差函数是：

> **生成器误差 = −ln(prediction)**

这个误差用来训练**生成器**的权重。

### 公式表示

设 `G(z)` 为生成器的输出，`D(G(z))` 为判别器对生成图像的输出：

- **生成器误差** = −ln( D(G(z)) )
- **判别器误差** = −ln( 1 − D(G(z)) )

这两个导数分别用于更新两个网络的权重，以改进该次预测。

> 注意：这两个误差函数**相互对抗**，但没关系——
> - 红色（生成器）误差函数**只**改变生成器权重；
> - 蓝色（判别器）误差函数**只**改变判别器权重。
>
> 因此它们不会冲突，而是把两个网络**作为一个整体一起改进**，产生不同的输出。这正是 GAN 的精妙之处。

## 反复迭代

接下来把这个过程重复很多次：

1. 随机取一个 z 值。
2. 用生成器产生一张假图。
3. 用判别器判断这张假图。
4. 用反向传播**同时更新**生成器和判别器的权重。
5. 再取一张**真实图像**，送入判别器，再次用反向传播更新（判别器）权重。

经过许多次迭代（**epoch**）后会怎样呢？我们在 notebook 里跑完训练，得到了一组权重（图中**粗边表示正权重，细边表示负权重或零**）。

可以暂停视频，自行验证这两个网络都工作得很好：

- 生成器能生成逼真的 Slanted Land 人脸；
- 判别器能区分人脸与非人脸。

### 验证生成器

回顾开头：人脸的左上、右下角应大，另外两角应小。

- 输入在 0–1 之间，且上方两条边为正权重 → sigmoid 输出大 → **左上像素值大**。
- 同理另两个正权重 → **右下像素值大**。
- 两个负权重 → **右上像素值小**。
- 另两个负权重 → **左下像素值小**。

因此生成的图像很像一条对角线。在 Slanted Land 的 2×2 分辨率下，这张图看起来就像当地的一位居民。**我们成功构建了一个生成人脸的网络。**

## 代码（Sing Along）

如承诺所言，代码在 GitHub 仓库 **GANs** 中。主要内容：

- **数据**：硬编码的人脸图像，以及随机噪声（不一定是人脸的）图像。
- **Discriminator 类**：仔细推导并实现了判别器的导数——分别针对人脸和噪声图像，且对**权重**和**偏置**都求了导。
- **Generator 类**：实现了生成器误差函数对应的导数，同样对**权重**和**偏置**求导。
- **误差函数曲线**：可以绘制生成器和判别器的误差曲线。
  - 生成器误差**下降并趋于稳定**；
  - 由于生成器在追赶判别器，判别器误差表现没那么好，**最后还略有上升**。
- **生成结果**：让训练好的生成器生成一些随机图像，它们看起来都像 Slanted Land 的人脸——正是我们想要的。

至此，我们成功创建了一对能在 Slanted Land 生成人脸的 GAN。
