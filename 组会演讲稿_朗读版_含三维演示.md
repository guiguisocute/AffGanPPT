## 0　开场白

【幻灯：封面】

大家好，今天我给大家精读一篇 ICLR 2017 的 Oral 论文，《Amortised MAP Inference for Image Super-Resolution》，中文可以叫《用于图像超分辨率的摊销 MAP 推断》，一般用它提出的模型名字简称 **AffGAN**。作者来自 Twitter 伦敦团队和哥本哈根大学，其中 Ferenc Huszár 大家可能在博客圈见过。

这篇论文回答的问题只有一个：**超分辨率为什么会出糊图，以及怎么从数学根子上把它治好**。今天我除了幻灯，还准备了三个可以交互的三维演示页面，关键概念我会现场拖给大家看。

## Part 1　背景与动机（约 10 分钟，含演示 ①）

【翻页 → Part 1 扉页，一句带过】

第一部分，背景与动机：为什么要放弃 MSE、转向 MAP。

【翻页 → 与 PRSR 的区别】

先接上我们组之前讲过的旧知识。上次我们学了 Pixel Recursive SR，它是**显式建模条件分布，然后逐像素采样**——慢，但能给出多样的解。今天这篇走的是另一条路：**不显式计算图像密度，训练一个前馈网络，一次前向直接输出一个近似 MAP 解**——快，但每个输入只有一个答案。两条路在论文 5.6 节还会重新交汇，今天最后也会回到这个对比。

【翻页 → 扫盲·何为点估计】

在讲 MAP 之前，先扫个盲：什么叫"点估计"。贝叶斯方法给我们的其实是**一整条后验分布**——

**公式**：$p(y|x)$
**读作**：「p，y 给定 x」；含义是"在观测到低清图 x 的条件下，高清图 y 的后验概率"。

它包含了所有可能的高清解，以及每个解的概率。但是产品落地时只能输出**一张确定的图**，所以必须从这条分布里挑一个"代表点"——这个动作就叫**点估计**。挑哪个点？这就是下一页的内容。

【翻页 → 点估计三兄弟】

点估计有"三兄弟"：**均值、中位数、众数**。关键结论是：**你训练时选的损失函数，已经在数学上替你决定了网络会输出三兄弟中的哪一个**。用平方损失、也就是 MSE 训练，贝叶斯最优解是后验**均值**；用绝对值损失、也就是 MAE，最优解是后验**中位数**；用零一损失，对应的是后验**众数**，也就是 MAP——

**公式**：$\hat y = \operatorname*{argmax}_y\, p(y|x)$
**读作**：「y 帽，等于 arg max，取使 p y 给定 x 最大的那个 y」；y 上面的小帽子表示"估计值"。

顺带把两个容易混的词分清：**MLE** 最大似然，只看似然项、不带先验，等价于"平坦先验下的 MAP"；**后验**等于似然乘先验再归一化。本文的主角自始至终是 MAP。

【翻页 → Figure 1 瑞士卷】

现在看论文的核心图，Figure 1，瑞士卷玩具实验。真实图像有几百万维，画不出来，作者就造了一个只有**两个像素**的世界：一张"图"就是 y 等于 y1、y2 两个数，也就是平面上一个点；"真实图像的分布"被人为设定成瑞士卷形状；下采样定义为两个像素取平均——

**公式**：$x = \dfrac{y_1 + y_2}{2}$
**读作**：「x 等于二分之一乘 y1 加 y2」。

给定一个观测，比如 x 等于零点五，所有合法的高清解构成一条橙色直线。后验就是拿这条线去"切"瑞士卷。图上标了三种策略的落点：MSE 的均值掉在螺旋带**之间的空隙**里，MAP 的众数稳稳落在**卷上**。右边的 Table 1 用数字盖章，这个表我们 Part 3 再细读。

这一页信息量很大，与其在平面图上比划，不如直接看立体的。

【翻页 → MAP 为何以前少用】

看完演示，一个自然的疑问：MAP 听起来这么合理，为什么以前很少用？两道坎。第一道，**先验难建**：MAP 需要知道"真实图像长什么样"，也就是几十万到百万维的图像先验，极难建模。第二道，**优化难解**：即便有了先验，还要对每一张输入图解一个非凸、多峰的约束优化问题，太慢。所以大家宁可用 MSE——简单、稳定、PSNR 好看，代价就是糊。

本文的三招正好破这两道坎：**摊销**，训练一个网络代替逐样本优化，破"慢"；**仿射投影层**，把约束变成结构强制，破"约束"；**GAN 或去噪梯度**，不用显式建模先验也能把输出推向高概率区，破"先验难建"。这三招就是 Part 2 的展开顺序。

## Part 2　方法（约 15 分钟，含演示 ②）

【翻页 → Part 2 扉页，一句带过】

第二部分，方法。这部分数学最密，我们配合第二个三维页面来讲。

【翻页 → 摊销 MAP 推断】

先说"摊销"这个词。传统 MAP 是每来一张低清图，就现场解一次优化；**摊销 MAP** 是把这个代价"摊"进训练——训练一个网络一次性学会对**所有** x 输出 MAP 解，测试时前向一次直接出图。理论出发点是贝叶斯分解：

**公式**：$\log p(y|x) = \log p(x|y) + \log p_Y(y) - \log p_X(x)$
**读作**：「log p，y 给定 x，等于 log p，x 给定 y，加 log p Y y，减 log p X x」；即"对数后验 = 对数似然 + 对数先验 − 一个与 y 无关的常数"。

三项里，难点在中间的**图像先验**和似然里藏着的**一致性约束** A y 等于 x。接下来两页分别处理。

【翻页 → 扫盲·零空间伪逆】

处理约束需要三个线性代数词汇：**列空间**——矩阵 A 能"打到"的所有方向；**零空间**——被 A 压成零的方向，沿着它走，下采样结果**不变**；**伪逆**——

**公式**：$A^+$，满足 $AA^+A = A$
**读作**：「A 的伪逆」，写作 A 右上角一个加号；性质读作「A 乘 A 伪逆乘 A，等于 A」。

这三个词是下一页核心公式的全部零件。与其干讲，我直接演示。

> 【切回幻灯】

【翻页 → 核心·仿射投影层】

把刚才看到的写成公式，这就是本文最硬核的创新，论文式 7：

**公式**：$g_\theta(x) = \Pi_x^A f_\theta(x) = (I - A^+A)\,f_\theta(x) + A^+x$
**读作**：「g 西塔 x，等于投影算子派、上标 A 下标 x，作用在 f 西塔 x 上；等于，括号，I 减 A 伪逆乘 A，括回，乘 f 西塔 x，再加 A 伪逆乘 x」。其中 θ 读"西塔"，是网络参数；I 是单位矩阵；Π 读"派"，表示投影。

两项各司其职：**A 伪逆 x** 是基线解，低频全对但没细节，刚才演示里的"大平图"；**I 减 A 伪逆 A** 把网络输出投进零空间，这部分无论加多少，下采样后恒为零。验证一行就够：A 乘 g，等于 A A 伪逆 x 加零，等于 x，**精确成立**。于是似然项被结构恒满足，约束优化变成了无约束优化。

【翻页 → 化归·交叉熵】

约束消掉之后就是全文最漂亮的一步，论文式 9：

**公式**：$\operatorname*{argmax}_\theta\, \mathbb{E}_x \log p_Y\big(\Pi_x^A f_\theta(x)\big) = \operatorname*{argmin}_\theta\, H[q_\theta,\, p_Y]$
**读作**：「arg max over 西塔，对 x 取期望，log p Y，投影后的输出；等于，arg min over 西塔，q 西塔与 p Y 的交叉熵」。E 读"对 x 取期望"；H 方括号里两个分布，读"前者相对后者的交叉熵"。

左边是摊销 MAP 目标：让所有输出的平均对数先验最大。右边说这**恰好等价于**最小化"模型输出分布"与"真实图像分布"之间的交叉熵——其中 q 西塔是把随机低清图喂给网络后、输出图像形成的分布。这一步的重大意义：**训练不再需要成对的高清-低清图像对**，超分从回归问题摇身变成了生成式建模问题——这正是接下来能搬出 GAN、去噪器、密度模型三种武器的原因。

【翻页 → 扫盲·熵与 KL】

后面要用到三个信息论概念，用"编码要花几个比特"来记：**熵** H q，用 q 自己的最优编码表，平均码长；**交叉熵** H q p，实际分布是 q、却错用了 p 的编码表，平均码长；**KL 散度**，错用编码表**多花**的那部分。三者的恒等式：

**公式**：$H[q, p] = H[q] + \mathrm{KL}[q\,\|\,p]$
**读作**：「q 相对 p 的交叉熵，等于 q 自身的熵，加上，q 到 p 的 KL 散度」；KL 方括号中间的双竖线，读"到"或"相对于"。

请记住这个式子，四页之后它会立大功。

【翻页 → 三种解法】

交叉熵还是没法直接算——老问题，真实图像的先验写不出来。论文给出三件武器：**AffGAN**，用判别器隐式地做；**AffDG**，用去噪器估计密度梯度；**AffLL**，直接训一个显式密度模型。三者共同起点都是仿射投影后的输出分布，区别只在近似手段。剧透一下结论：真实图像上 **AffGAN 视觉效果最好**，后两者偏糊。

【翻页 → GAN 运用 vs SRGAN】

GAN 基础大家都学过，本文两处改造：第一，生成器的输入**不是噪声 z，而是低清图 x**；第二，用**修正的生成器更新规则**，可以证明它最小化的是 KL 散度 q 西塔到 p Y。和大家熟悉的 SRGAN 对比：SRGAN 靠成对数据加 VGG 内容损失，一致性没有硬保证；AffGAN 用投影层**硬保证**一致性，原则上不需要成对数据，并且有明确的概率解释——它是 MAP 目标的替代。

【翻页 → 方法一 AffGAN】

AffGAN 的完整训练流程看这张图：采一批低清图过生成器，输出天然满足一致性；采一批**不相关的**真实高清图；两边都加实例噪声；更新判别器；再按修正规则更新生成器。这一套的动态效果，等 Part 3 开头我用第三个演示页现场直播。

【翻页 → 交叉熵 vs KL 彩蛋】

刚才让大家记住的恒等式，现在立功了。移项：

**公式**：$H[q_\theta, p_Y] - \mathrm{KL}[q_\theta\,\|\,p_Y] = H[q_\theta]$
**读作**：「交叉熵减 KL 散度，等于 q 西塔自身的熵」。

我们**想要**最小化的是交叉熵；GAN **实际**最小化的是 KL。两者差了一项——q 西塔自己的熵。也就是说，GAN 比纯 MAP 目标**多奖励了输出分布的熵**：鼓励不同输入给出多样的输出、惩罚所有 x 挤到同一个答案。这是一个有益的副作用，天然抗模式塌缩。

【翻页 → 实例噪声 Figure 6】

GAN 训练不稳的病根：真实分布和生成分布都高度集中在低维流形上，支撑集几乎不相交，判别器能用无数种方式把它们完美分开，给生成器的梯度方向就乱了。**实例噪声**的药方：给真假样本**都**加同强度的高斯噪声——

**公式**：$d_\sigma = \mathrm{KL}[\,p_\sigma * q_\theta\,\|\,p_\sigma * p_Y\,]$
**读作**：「d 西格玛，等于，p 西格玛卷积 q 西塔，到，p 西格玛卷积 p Y 的 KL 散度」；星号读"卷积"，σ 读"西格玛"，是噪声标准差。

两团分布被"吹胖"后产生重叠，最优判别器变唯一、梯度变稳，然后把 σ 随训练**线性退火**到零。论文附录还证明：和单边标签平滑不同，实例噪声**不会使最优判别器产生偏差**。

【翻页 → 扫盲·去噪=梯度】

第二件武器之前，再补一个关键洞见：**去噪等于学习对数密度的梯度**。

**公式**：$\dfrac{f^*(\tilde y) - \tilde y}{\sigma^2} \approx \nabla \log p_Y(\tilde y)$
**读作**：「f 星，y 波浪，减 y 波浪，除以西格玛平方，约等于，log p Y 的梯度」；f 星是贝叶斯最优去噪器，y 波浪是加了噪声的样本，∇ 读"梯度"或"nabla"。

直白说：训练一个去噪器，"去噪输出减带噪输入"这个残差，就近似指向密度升高最快的方向。不用写出先验的密度函数，也能知道"往哪走更像真图"。大家应该已经听出来了——这就是今天 score-based 扩散模型的思想源头之一。

【翻页 → 方法二 AffDG】

AffDG 就是把这个洞见用起来：预训练一批不同噪声水平的去噪自编码器，把它们给出的梯度按链式法则反传进超分网络，让输出沿着对数先验爬坡；噪声从大到小退火。论文特意说，据他们所知，这是**首次**把去噪器的输出显式反传去训练另一个网络。

【翻页 → 方法三 AffLL】

第三件武器最直接：用最大似然训一个显式密度模型当先验打分器，最小化输出的负对数似然。模型用类 PixelCNN 的结构——注意这正是和我们上次学的 PRSR 的接口——但原始 PixelCNN 的离散似然不可微，本文换成了连续可微的 MCGSM。区别在于：AffLL **不逐像素采样**，只拿密度当打分器。实测偏糊，原因 Part 4 分析。

## Part 4　总结与讨论（约 5 分钟）

【翻页 → Part 4 扉页，一句带过】

最后一部分，总结与讨论，包括作者对自己方法的批评。

【翻页 → 批评·高维众数】

作者非常清醒地指出 MAP 的两个隐患。第一，**众数依赖表示**：换一个色彩空间再做 MAP，答案可能不一样。第二，也是更深刻的，**高维下众数未必典型**：由于测度集中现象，d 维标准高斯的典型样本范数约等于根号 d，而众数在原点、范数为零——众数是**高度非典型**的点。这个"高斯肥皂泡"的比喻就出自作者 Huszár 自己的博客。所以纯 MAP 的"高概率"不等于"看起来真实"，这为下面的采样扩展埋下伏笔。

【翻页 → 补充·Figure 7】

为什么 AffDG 和 AffLL 收敛了却糊？图 7 显示两个模型都在正常收敛。AffDG 的问题是噪声退火两难：σ 大，梯度方向粗但覆盖广；σ 小，梯度准但只在流形附近有效。AffLL 的问题是：密度模型本身不差——负四点一比特每维——但真实数据流形附近密度**过于陡峭**，打分不够精确，训练早期极难。这解释了为什么高维真实图像上 AffGAN 是赢家。

【翻页 → 变分推断】

未来方向，附录 F：给生成器额外喂一个噪声 z，输出就从"一个点"变成"一个分布"，论文证明随机版 AffGAN 实际在做**摊销变分推断**——最小化近似后验到真实后验的 KL 散度，可以对同一张低清图采出多张一致且多样的高清图。

【翻页 → 总结】

一页回顾整条主线：**MAP** 定目标，取众数不取平均；**仿射投影**消似然，合法性焊死在结构里；**交叉熵**化归目标，顺便甩掉配对数据；**三种解法**里 GAN 胜出。和 PRSR 的呼应：PRSR 逐像素采样后验，慢但多样；AffGAN 一步输出众数，快但唯一——同一个贝叶斯问题的两种取舍。

## Q&A 备答（不念，供应对提问）

**问题一：仿射投影能否接到现有 SR 模型上？**
可以，条件有三：退化算子 A 已知、线性、无噪声，且伪逆可数值求出。它就是一个固定线性层，加在任意 SR 网络末端即可，训练时冻结。边界也在这：真实退化含噪声或未知模糊核时，严格等式约束不再合理——这正是它和 SoftGAN 软惩罚的本质分工：软约束假设有观测噪声、只能近似满足；仿射投影假设无噪声、严格满足。

**问题二：众数 vs 采样，怎么选？**
看下游用途。只出一张图给人看、要快——选众数（AffGAN 路线）；需要不确定性量化、多候选（医学影像、侦察判读）——选采样（PRSR 或附录 F 的随机 AffGAN）。理论上注意高维众数非典型问题：众数密度最高但统计上"不典型"。

**问题三：实例噪声 vs 扩散模型的加噪调度？**
共同点：都用高斯噪声"吹胖"数据分布获得重叠支撑集与良好梯度，都做由大到小的噪声调度；数学上都与"平滑后的分布"上的 score 有关。根本区别：实例噪声只是**训练期的正则化技巧**，噪声退火到零后模型是一步前馈生成；扩散模型把加噪/去噪本身作为**生成过程**，推理时要沿噪声阶梯逐步采样。可以说实例噪声（2016）是"把噪声当拐杖"，扩散（2020 起）是"把噪声当路"。

**可能的追问：交叉熵为什么在真实图像上算不了？**
H[q,p_Y] 的蒙特卡洛估计需要对任意生成样本求 log p_Y——真实图像的 p_Y 没有解析式；任何密度模型给的都是近似值（AffLL 的失败正说明近似密度打分不可靠）。瑞士卷实验的存在意义就是：那是唯一能拿"真密度"验证理论的地方。

**可能的追问：修正的生成器更新规则是什么？**
标准 GAN 生成器最小化 −log D；本文改为最大化 log(D/(1−D))（论文式 10），配合最优判别器 D* = p_Y/(p_Y+q_G)，可证生成器梯度方向等价于最小化 KL[q_θ‖p_Y]（附录 A）。
